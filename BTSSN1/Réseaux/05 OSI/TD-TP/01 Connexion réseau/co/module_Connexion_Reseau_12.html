Trame Ethernet

Le bloc d'information ou trame au niveau de la sous-couche MAC défini par la norme 802.3 est composé de huit champs :

  • 8 octets indiquant le début de la trame (préambule et délimiteur) ;

  • L'adresse MAC du destinataire de la trame ;

  • L'adresse MAC de l'émetteur de la trame ;

  • le protocole de la couche supérieure ;

  • les données avec éventuellement du bourrage ;

  • le code FCS qui permet de détecter d'éventuelle erreur de transmission.

Complément :

Le préambule est composé de 7 octets formés d'une succession de 0 et de 1, assure la synchronisation du récepteur sur la trame émise (horloge). Il est suivi par le délimiteur de début de trame (octet 10101011) qui permet de trouver le début du champ des adresses.

Le champ de « bourrage » n'est utilisé que si la longueur de la trame est inférieure à la limite imposée pour le fonctionnement correct de l'algorithme CSMA/CD (la taille totale des champs « bourrage » et « information » doit être supérieure ou égale à 46 octets).

La somme de contrôle (Frame Check Sequence) sur 4 octets est calculée suivant un code de redondance cyclique (CRC) qui permet de détecter les erreurs de transmission (polynôme cyclique g(x) : g(x) = x32 + x26 + x22 + x16 + x12 + x11 + x10 + x8 + x7 + x5 + x4 + x2 + x).
