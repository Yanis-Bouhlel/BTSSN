Les réponses HTTP

Format d'une réponse

Une réponse HTTP se compose de :

Une ligne de statut ;

Les champs d'en-tête de la réponse facultatifs ;

Le corps de la réponse c'est-à-dire le CONTENU du fichier.

Une réponse HTTP a donc la syntaxe suivante (\r\n sont les caractères retour chariot et saut de ligne) :

Syntaxe

VERSION-HTTP CODE EXPLICATION\r\n

EN-TETE : Valeur\r\n

...

EN-TETE : Valeur\r\n

\r\n

CORPS DE LA REPONSE

Codes retour

Les codes retour sont importants car ils représentent le statut de la transaction. Le code de réponse est constitué de trois chiffres, le premier indique la classe de statut et les suivants la nature exacte de l'erreur.

Exemples : 200 pas d'erreur et 404 page non trouvée

Exemple

Exemple de réponse

HTTP/1.1 200 OK

Date: Sun, 27 Feb 2011 07:01:07 GMT

Server: Apache/ProXad [Aug 9 2008 02:45:09]

Cache-Control: no-store, no-cache, must-revalidate, post-check=0, precheck=0

Expires: Thu, 19 Nov 1981 08:52:00 GMT

Pragma: no-cache

X-Powered-By: PHP/5.1.3RC4-dev

Connection: close

Content-Type: text/html

<!DOCTYPE html>

...

Complément

La réponse HTTP/1.1 du serveur au client commence par le statut suivi par l'entête et le corps de l'entité.

La ligne de statut comprend trois éléments séparés par un espace :

La version du protocole utilisé ;

Le code de statut ;

La signification du code.

La ligne de statut précise la version du protocole utilisée et l'état du traitement de la requête à l'aide d'un code et d'un texte explicatif. Le code est utilisé par le programme client alors que l'explication permet d'informer l'utilisateur en cas d'erreur.

Les champs à connaître ne sont pas nombreux. Vous pouvez vous en tirez très bien avec seulement les quatre informations suivantes :

Date : indique la date de début de transfert des données.

Content-Type : indique le type MIME du corps de la réponse.

Content-length : indique la taille en octets du corps de la réponse.

Location : indique une redirection.
