Le datagramme IP

Le datagramme IP est constitué de deux zones :

  • une zone « datagramme header » (entête) ;

  • une zone « datagramme data » (données).

Chacune des zones est de longueur variable.

Le datagramme est constitué d'un ensemble de mots de 32 bits pour faciliter la vitesse de transfert, le stockage et le calcul dans les stations. Donc le format du datagramme est toujours affiché non pas octet par octet mais par des mots de 32.

Format de l'entête IP

Version : Le champ Version codé sur 4 bits permet de distinguer entre les différentes versions du protocole IP. Actuellement, les versions ne prennent que deux valeurs :

  • 4 : IPV4, version 4, adresse sur 32 bits

  • 6 : IPV6, version 6, adresse sur 128 bits

IHL (Internet Header Length) : Le champ Longueur de l'entête ou IHL codé sur 4 bits permet de calculer le nombre de mots de 32 bits (4 octets) de la taille de l'en-tête et des options du datagramme IP. Si il n'y a pas d'option, il est égal à 5 soit 20 octets.

Type de service : Le champ Type de service est codé sur 8 bits. Ce champ contient des informations qui aideront les routeurs à choisir le chemin pour le paquet en indiquant l'importance du datagramme.

Longueur totale : Ce champ est codé sur 16 bits, il contient la taille totale en octets du datagramme. Utilisé avec la longueur de l'en-tête (IHL), il permet de déterminer où commence exactement les données transportées.

Identification : Le champ Identification, codé sur 2 octets, est défini de manière distincte pour chaque datagramme et contient le numéro du datagramme C'est l'émetteur qui numérote le datagramme. Si le fragment doit être fragmenté, tous les fragments porteront ce numéro.

Protocole : Le champ Protocole, codé sur 1 octet, indique le protocole de la couche supérieure transporté par le paquet. Une liste des protocoles de niveau supérieur se trouve dans le RFC 1700. Les valeurs les plus couramment rencontrées sont :

  • 1 : ICMP : Internet Control Message protocol

  • 2 : IGMP : Internet Group Message Protocol

  • 6 : TCP : Transmission Control Protocol

  • 17 : UDP : User Datagram Protocol

Contrôle d'en-tête : Ce champ est une somme de contrôle (Checksum) codée sur 2 octets, il ne sert que pour vérifier qu'il n'y a pas d'erreur dans l'en-tête IP.

Adresses IP source et destination : Les champs Adresse IP source et destination sont les adresses IP, codés sur 4 octets, des nœuds source et destination.
