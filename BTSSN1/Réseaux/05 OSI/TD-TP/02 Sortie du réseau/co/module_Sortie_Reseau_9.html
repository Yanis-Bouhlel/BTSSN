Fragmentation

Les datagrammes vont circuler sur des réseaux différents avant d'arriver au destinataire. Malheureusement la longueur des trames circulant dans chaque réseau a généralement une taille maximale différente dépendant de leur constitution matérielle. Ces tailles s'appellent MTU (Maximum Transfert Unit).

Il ne faut pas confondre la fragmentation des paquets et le fractionnement du flux de données.

Lorsqu'un hôte envoie un flot de données (un fichier par exemple) dans un réseau, il doit préalablement le fractionner en morceaux. Ce fractionnement se fait au niveau de la couche de transport donc niveau 4.

La fragmentation est mise en œuvre par des routeurs car ils sont seuls à pouvoir connaître les caractéristiques matérielles des réseaux auxquels ils sont reliés. Si un datagramme IP dépasse la taille de MTU du réseau sur lequel le routeur a choisi d'envoyer le datagramme, il doit être découpé en morceaux plus petits. On appelle cela la fragmentation.

Chaque fragment du datagramme originel est envoyé sous forme de datagramme IP. L'entête IP contient des informations permettant d'identifier le fragment. Ces informations seront utilisées par l'hôte de destination pour ré-assembler les fragments.

Les routeurs n'assemblent pas les datagrammes fragmentés. Cette opération est implémentée par le nœud de destination.

Paramètres de fragment

Les paquets d'informations fragmentés s'éparpillent dans le réseau et peuvent arriver dans le désordre, il est donc nécessaire d'indiquer sur chaque datagramme de quel morceau il s'agit.

Ces indications sont portées par le champ de l'entête IP composées des bits 0 - DF - MF et du mot de 13 bits "Décalage de fragment".

Champ

Information

DF

Don't Fragment : ne pas Fragmenter ce paquet

MF

More Fragments : ce datagramme n'est pas le dernier fragment

Le champ Drapeau peut donc prendre les valeurs suivantes :

  • 001 : La fragmentation est autorisée, le fragment n'est pas le dernier

  • 000 : La fragmentation est autorisée, le fragment est le dernier

  • 010 : La fragmentation n'est pas autorisée

Place du fragment

Ce champ indique la position relative des données du fragment par rapport au début du datagramme initial. Ce décalage est 0 pour le premier fragment puisqu'il contient le début de l'ensemble des données à transmettre. Le nombre écrit dans le champ Décalage de fragment indique non pas le nombre d'octets mais le nombre de mots de 8 octets.

Ainsi, si ce champ contient le nombre 100 le décalage indiqué sera de 800 octets. Lors de la fragmentation, il faudra donc faire en sorte que la longueur des données du datagramme fragmenté soit un multiple de 8 octets.
