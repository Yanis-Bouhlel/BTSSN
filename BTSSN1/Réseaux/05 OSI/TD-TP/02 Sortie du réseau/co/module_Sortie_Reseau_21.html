DNS

Prenons pour exemple, un ordinateur du réseau de la section SNIR qui a pour adresse 192.168.0.10. Si nous lançons un navigateur type Chrome ou Firefox avec l'affichage de la page "www.google.fr". Que se passe-t'il ?

La première étape est de déterminer l'adresse IP de "www.google.fr". Il faut donc la demander au serveur DNS, cela signifie :

  1. Le serveur DNS a pour adresse IP 192.168.0.2 donc son adresse réseau est 192.168.0.0 (application du masque réseau). L'adresse réseau de l'ordinateur est aussi 192.168.0.0 (192.168.0.10 ET 255.255.255.0). Donc le serveur DNS est dans le même réseau.

  2. Une requête ARP est donc envoyée sur le réseau avec pour adresse IP celle du serveur DNS et pour adresse MAC, l'adresse de broadcast (FF:FF:FF:FF:FF:FF).

  3. Le serveur DNS renvoie une réponse ARP avec son adresse MAC.

  4. L'ordinateur peut envoyer maintenant une requête au serveur DNS pour obtenir l'adresse IP de "www.google.fr".

  5. Le serveur va demander à un des serveurs racine DNS, l'adresse IP d'un serveur DNS qui gère la zone "fr".

  6. Puis le serveur DNS demandera au serveur DNS de la zone "fr", l'adresse de serveur DNS de la zone "google.fr".

  7. Enfin le serveur DNS demandera au serveur DNS de la zone "google.fr", l'adresse IP de la machine "www.google.fr".

  8. Le serveur DNS répondra à l'ordinateur 192.168.0.10 et lui enverra l'adresse IP de "www.google.fr".

Maintenant le navigateur peut demander au serveur Web la page Web.
