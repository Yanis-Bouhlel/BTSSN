Points importants

Fondamental : Page envoyée

Important : dans le navigateur, vous ne précisez pas en général le nom de la page à envoyer. En effet, la page affichée est en général la page "index.html".

Si ce fichier n'existe pas, les fichiers index.cgi, index.pl, index.php, index.xhtml ou index.htm sera affiché (avant interprétation si nécessaire). Si aucun de ces fichiers n'est présent, la liste des fichiers est affichée.

Codage par défaut des pages Web

L'encodage des pages Web est souvent un problème. En cas de problème, les caractères accentués ne sont pas bien affichés.

Vous pouvez remplacer tous les caractères spéciaux par un code commençant par & (esperluète ou « et commercial ») et terminant par ; (point virgule).

Vous pouvez aussi préciser le type d'encodage de votre page par une balise meta, par exemple en HTML5 pour préciser l'encodage UTF-8 :

<meta charset="UTF-8">

Les encodages les plus fréquents (pour le public français) sont :

  • l'ISO-8859-1 (parfois appelé latin1), qui permet d'enregistrer presque tous les caractères du français ;

  • l'ISO-8859-15 (parfois appelé latin9), une variation de l'ISO-8859-1, qui rajoute le symbole « euro » et le « l'o dans l'e» ;

  • l'UTF-8, qui permet théoriquement d'encoder toutes les langues, du français au japonais en passant par l'arabe.
