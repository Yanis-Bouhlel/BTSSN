Les classes d'adresses IP

Les adresses IP sont réparties en classes appelées A, B, C, D et E selon le nombre d'octets qui représente le réseau. Pour distinguer les classes A, B, C, D et E, il faut examiner les bits de l'octet de poids fort.

Adresses IP de classe A

Si le bit le plus significatif de celui-ci est à « 0 », l'adresse IP est de classe A.

On dispose donc de 7 bits pour identifier les réseaux et de 24 bits pour identifier les hôtes ce qui permet d'avoir 126 réseaux (1.0.0.0 à 126.0.0.0) et 16 777 216 hôtes par réseau. Il est à remarquer que le réseau 0 n'est pas utilisé, il a une signification particulière « tous les réseaux ».

De même, l'adresse hôte 0 n'est pas utilisée, tout comme l'adresse ayant le plus fort numéro dans le réseau (tous les bits de la partie l'adresse hôte sont à 1).

Les adresses possibles vont donc de 1.x.x.x à 126.x.x.x (les adresses 127.x.x.x sont réservées).

Adresses IP de classe B

Si les deux premiers bits les plus significatifs sont à « 10 », l'adresse est de classe B.

On dispose donc de 14 bits pour identifier les réseaux et de 16 bits pour identifier les hôtes. Ce qui fait 214 = 16 384 réseaux (128.0.0.0 à 191.255.0.0) et 65 534 hôtes (65 536 - 2) par réseaux.

Les adresses possibles vont donc de 128.0.x.x à 191.255.x.x.

Adresses IP de classe C

Si les trois bits les plus significatifs sont à « 110 », l'adresse sera de classe C.

On dispose donc de 21 bits pour identifier les réseaux et de 8 bits pour identifier les hôtes. Ce qui fait 221 = 2 097 152 réseaux (de 192.0.0.0 à 223.255.255.0) et 254 hôtes (256 - 2).

Les adresses possibles vont donc de 192.0.0.x à 223.255.255.x.

Adresses IP de classe D

Pour pouvoir transmettre des informations à plusieurs systèmes, il faut utiliser une adresse multicast (diffusion). Les adresses de cette classe sont utilisées.

Adresses IP de classe E

Ce sont des adresses réservées pour un usage ultérieur.
